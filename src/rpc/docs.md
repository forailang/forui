# Forui.rpc

`Forui.rpc` is the server-side support module for Forai `remote def` calls. The
compiler discovers reachable remote functions, generates RPC dispatch code, and
registers the `/fai/rpc` and `/fai/interface` routes through the generated
`addRpcRoutes` function.

Most fullstack apps should use the generated route installer from the server
entry point:

```fai
use std.http.server

use { getTasks, addTask } from data.tasks

def main
    @return Void
do
    var r = server.router()
    addRpcRoutes(r)
    server.listen(r, 3040)
end
```

`addRpcRoutes` is generated by the build from reachable `remote def`s. Do not
define it manually. Import the modules that contain remote functions from the
server target so they are part of the reachable build graph.

## Remote Functions

Remote functions and remote types live in normal app modules. Client targets
import and call them from their real module path; the build replaces those calls
with generated RPC stubs for client code.

```fai
remote type Task
    id Int
    text String
    done Bool
end

remote def getTasks
    @return Task[]
do
    loadTasks()
end

remote def addTask
    @param text String
    @return Task
do
    insertTask(text)
end
```

## Request Handling

`handleRpcRequest(request, specJson, specHash, dispatch)` is the core request
handler used by generated or custom RPC route wiring. It handles:

- `GET /fai/interface`, returning the generated interface spec JSON
- `POST /fai/rpc`, parsing the function name, arguments, and interface hash
- dispatching to the generated `(fnName, argsJson) -> String` function
- wrapping successful results with `rpcOk`
- returning RPC errors for unknown functions, hash mismatches, and missing routes
- attaching cookies queued by remote functions

Most app authors should let `addRpcRoutes(r)` call this through generated code.
Call `handleRpcRequest` directly only when building custom server routing.

## Cookies From Remote Defs

Use `queueCookie` inside a remote function when the RPC response should set or
clear a cookie without changing the remote function's return type. The RPC
handler drains pending cookies with `takePendingCookies` after dispatch and
merges them into the `HttpResponse`.

```fai
use { queueCookie } from Forui.rpc

remote def login
    @param email String
    @param password String
    @return Session?
do
    let session = createSession(email, password)
    if session?
        queueCookie(sessionCookie(session!.token))
    end
    session
end
```

## Protocol Helpers

Functions such as `parseFnName`, `parseArgs`, `parseHash`, `rpcOk`,
`rpcError`, `rpcHashMismatch`, `withCookies`, `takePendingCookies`, and
`serveRpc` are protocol or framework helpers. They are useful for tests,
generated code, and adapter work, but ordinary app code should not need them.

The request body sent by generated client stubs contains:

```json
{"fn":"module.function","args":[...],"hash":"interface-hash"}
```

`parseArgs` returns the arguments as a JSON string because generated dispatch
functions accept `(fnName: String, argsJson: String)`.
